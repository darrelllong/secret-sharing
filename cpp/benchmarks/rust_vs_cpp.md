
## Rust vs C++ — pilot-bench (k=3, n=5, GF(2^127 − 1))

| Operation                 | impl | ms/op    | ±CI (95%)  | Runs | C++ speedup |
|---------------------------|------|---------:|-----------:|-----:|------------:|
| `shamir_split`            | rust | 0.005800 | ±0.0004635 |  57  |             |
| `shamir_split`            | cpp  | 0.004483 | ±0.0005127 |  34  | 1.29×       |
| `shamir_reconstruct`      | rust | 0.007172 | ±0.0002041 |  42  |             |
| `shamir_reconstruct`      | cpp  | 0.005587 | ±0.0001882 |  30  | 1.28×       |
| `shamir_split_4kb`        | rust | 1.136    | ±0.02622   |  53  |             |
| `shamir_split_4kb`        | cpp  | 0.7486   | ±0.01653   |  35  | 1.52×       |
| `shamir_reconstruct_4kb`  | rust | 1.941    | ±0.05856   |  30  |             |
| `shamir_reconstruct_4kb`  | cpp  | 1.483    | ±0.04480   |  90  | 1.31×       |

Generated by `cpp/scripts/bench_compare.sh` (preset: quick, PILOT_SS_ITERS_PERCENT=100).

Both implementations share the same algorithms — limb-by-limb
schoolbook with the same Karatsuba threshold, identical Mersenne-127
fold, identical Lagrange interpolation — so the gap is in code-gen
and allocator behaviour:

- **Allocator.** `std::vector<u64>` on libc++ uses size-class bins
  via macOS's nano-zone for the small per-`big_uint` allocations
  (1–4 limbs); Rust's `Vec<u64>` goes through `std::alloc::System`
  which is `malloc`. For the 4 KiB-block ops (274 BigUint allocations
  per call) the per-allocation gap compounds, which is where the
  ratio peaks (1.52× on `shamir_split_4kb`).
- **Code-gen.** Apple clang vs `rustc` LLVM see slightly different
  inlining decisions through the schoolbook inner loop and the
  Mersenne fold in `mul_mod_mersenne127` — both are short enough to
  be sensitive to register-allocator quirks.

These are toolchain-level differences, not bit-compatibility
differences. `compat.shamir_share_bytes_match_rust` and
`compat.shamir_reconstruct_from_rust_shares` in `cpp/test/test_compat.cpp`
confirm the C++ port produces the same share bytes from the same seed
and reconstructs the same secret from Rust-produced share bytes.
